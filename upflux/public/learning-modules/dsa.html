Arrays & Basics Core

Arrays are contiguous memory blocks. Master indexing, traversal patterns, and time/space complexity.

  • Know \(O(1)\) access, \(O(n)\) search, and typical insert/delete costs.
  • Practice two-pointer, sliding window, prefix sums, and frequency counting.
  • Review edge cases: empty input, single element, duplicates, negative numbers.

Stack LIFO

Stacks model “last in, first out” behavior and are often used for backtracking and parsing.

  • Use for parentheses validation, next greater element, monotonic stack patterns.
  • Understand call stack + recursion relationship.
  • Practice converting infix ⇄ postfix, evaluating expressions.

Queue FIFO

Queues model “first in, first out” behavior. Variants power graph traversal and scheduling.

  • Know normal queue vs deque; when to use each.
  • Practice BFS, level-order traversal, sliding window maximum (deque).
  • Understand circular buffer implementation idea.

Linked List Pointers

Linked lists trade fast insertions for slower random access. Great for pointer manipulation practice.

  • Practice reversing (iterative + recursive), detecting cycles (Floyd).
  • Master dummy head, fast/slow pointers, merge two lists.
  • Be careful with null checks and pointer updates order.

Binary Search Divide & Conquer

Binary search finds an element (or boundary) in a sorted space by halving the search range.

  • Use low/high invariants; avoid infinite loops with mid calculation.
  • Practice boundary searches: first/last occurrence, lower/upper bound.
  • Apply to “answer space” problems (min/max feasible value).

Sorting Algorithms Fundamentals

Sorting improves search and enables greedy/2-pointer techniques.

  • Know trade-offs: quicksort avg \(O(n \log n)\), mergesort stable, heapsort in-place-ish.
  • Practice “sort then scan”: intervals, two-sum variants, grouping duplicates.
  • Understand stability and when it matters.

Recursion Mindset

Recursion solves a problem by reducing it to smaller subproblems until a base case.

  • Always define base case + progress toward it.
  • Trace using a small input; learn to visualize stack frames.
  • Practice tree recursion: permutations, subsets, DFS.

Binary Tree Traversals

Binary trees model hierarchical data. Traversals are the building blocks for most problems.

  • Know inorder/preorder/postorder, level-order (BFS).
  • Practice height/diameter, LCA, path sums.
  • Watch for null nodes and recursion depth.

Heap Priority

Heaps efficiently get min/max elements. Useful for top-k, scheduling, and streaming data.

  • Know push/pop \(O(\log n)\), peek \(O(1)\).
  • Practice top-k frequent, merge k sorted lists, running median (two heaps).
  • Understand heapify concept and comparator usage.

Hash Table Lookup

Hash tables give average \(O(1)\) lookup/insert and power counting + mapping patterns.

  • Practice frequency maps, two-sum, anagram grouping.
  • Know collision concept; prefer immutable keys.
  • Be mindful of worst-case and memory usage.

Graphs Modeling

Graphs represent relationships. Choose adjacency list for sparse graphs, matrix for dense graphs.

  • Identify directed vs undirected, weighted vs unweighted.
  • Practice connected components, shortest path basics.
  • Track visited carefully to avoid infinite loops.

DFS & BFS Traversal

DFS explores deep; BFS explores level by level. Both solve reachability and component problems.

  • BFS is ideal for shortest path in unweighted graphs.
  • DFS is great for cycle detection, topological ordering, recursion practice.
  • Practice grid problems (islands) using both approaches.

Dynamic Programming Optimization

DP stores results of overlapping subproblems to avoid repeated work.

  • Steps: define state, transition, base case, iteration order.
  • Practice 1D DP (stairs, house robber), knapsack patterns, LIS basics.
  • Try memoization first, then convert to tabulation.

AVL Tree Balanced BST

AVL trees keep height balanced via rotations to guarantee \(O(\log n)\) operations.

  • Understand balance factor and rotation types (LL, RR, LR, RL).
  • Know why balancing prevents worst-case skew.
  • Practice identifying rotation needed from imbalance pattern.

Trie Strings

Tries store prefixes efficiently and help autocomplete and prefix queries.

  • Practice insert/search, prefix search, word break style problems.
  • Know memory trade-offs vs hashing.
  • Track end-of-word markers properly.

Topological Sort DAG

Topological ordering arranges tasks with prerequisites (only works on DAGs).

  • Practice Kahn’s algorithm (in-degree queue) and DFS postorder.
  • Detect cycles when no node has in-degree 0 (or during DFS).
  • Use for dependency resolution and scheduling problems.
